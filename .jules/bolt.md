## 2024-05-19 - [Optimize targets loops]
**Learning:** High-frequency game loops (60FPS) were using intermediate array allocations (`.filter`) and higher-order functions (`.forEach`) on the `targets` array (which can be quite large). This caused garbage collection stutters. This confirms the specific requirement: "To maintain 60 FPS and prevent Garbage Collection (GC) stutters, avoid intermediate array allocations (e.g., using `.filter()`, `.every()`, or `.forEach()`) in high-frequency functions like `update()`, `draw()`, or `tap()`. Use standard `for` loops instead."
**Action:** Replaced `.forEach` with standard `for` loops for `targets` in `draw()` and `update()` loop. Replaced `targets.filter` with standard `for` loops that manually push elements into an array to avoid callback overhead in functions `getActiveSplitFamilyMembers` and `pruneInactiveSplitTargets`.
